Hexo 入门

发表于 2021-05-05 更新于 2021-05-07 分类于 前端 ， Node.js

简介

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

无需服务器，可以利用 Github、Gitee（码云）等托管在互联网中，实现网上部署自己的博客

部署到 Github

如果是在本地才能浏览到自己的博客，好像也没什么意思，但好在 Hexo 属于静态网页部署，无需服务器就可以在免费的静态网址托管平台上部署，利用 Github、Gitee、Goding 等

请确保当前网络环境可以正常访问 GitHub

准备

首先确认你已经有一个 Github账号

  • 新建一个 仓库 ，仓库命名规则为你的 用户名.github.io ，点击 create repository 即可

  • 跳转到新页面后，复制以 .git 结尾的仓库地址

  • 修改 _config.yml （在文件末尾）

  • 配置 git 账号和邮箱，运行到命令行中

  • 安装用于部署到 git 的插件

  • 远程部署

    在部署过程中需要依次填的 用户名 和 密码

  • 刷新查看仓库仓库是否成功上传

  • 访问 https://你的用户名.github.io

正常访问即部署成功

配置 SSH 提高效率

无论是创建新的文章还是更新文章，在进行远程部署时都需要输入用户名和密码，很是繁琐，而通过配置 SSH 密钥，就可以轻松解决这些不必要的烦恼

现在配置 Github 的 SSH 连接

  • 生成 SSH 密钥

    一路回车，注意保存的位置（C:\Users\用户名\.ssh 隐藏文件夹）

  • 打开 .ssh 文件夹，复制 id_rsa.pub 文件的内容

  • 点击 SSH and GPG keys 的 New SSH key

  • Title 随便输，Key 粘贴 id_rsa.pub 文件的内容

  • 点击 Add SSH key 即可

  • ssh git@github.com 命令行测试是否连接成功

再次新建文章或者更新文章时， hexo d 就十分轻松了
